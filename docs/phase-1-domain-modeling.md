# Phase 1: Domain Modeling & System Design Specification

## 1. Architecture Overview & Core Philosophy
The Turnaround Operations System acts as a non-blocking, asynchronous observer of airport ramp environments. The data layer is designed around absolute type-safety to prevent runtime operational anomalies during high-concurrency flight turnaround procedures.

## 2. Core State Machines (Enums)

### A. TaskStatus
Governs the strict transitions of individual frontline ground tasks.
* **LOCKED**: Prerequisite tasks are incomplete. Ground crews cannot execute.
* **UNLOCKED**: Background dependencies are cleared. Safe for field execution.
* **COMPLETE**: Standard successful completion by field agents.
* **CONFLICT_COMPLETED**: Forced administrative bypass logged cleanly despite unfulfilled dependencies.

### B. FlightStatus
Tracks the real-world, physical chronological lifecycle phases of an aircraft at the station.
* **SCHEDULED**: Flight tracked in system prior to arrival.
* **IN_FLIGHT**: Aircraft actively en route to the station.
* **ARRIVED**: Plane parked at the gate; turnaround window is active.
* **DEPARTED**: Turnaround complete; aircraft has pushed back and custody has terminated.

## 3. Database Entity Schemas (MongoDB Atlas Mappings)

### Task Entity (`tasks` collection)
Maps execution dependencies and crew assignments.
* `id`: Immutable MongoDB ObjectId string.
* `taskId` / `flightId`: Direct tracking relations.
* `status`: Strongly typed `TaskStatus` enum string.
* `dependencies`: Array of prerequisite Task IDs.
* `startTime` / `endTime`: Chronological execution markers (`LocalDateTime`).

### Flight Entity (`flights` collection)
Binds the physical asset footprint to schedule data.
* `flightNumber`: Used as the natural primary key (`_id`) to ensure unique operational profiles.
* `gate`: Physical station tracking asset.
* `scheduledArrival` / `scheduledDeparture`: Temporal boundaries mapped as `LocalDateTime`.
* `status`: Strongly typed `FlightStatus` lifecycle enum.

### SupervisorReview Entity (`supervisor_reviews` collection)
Acts as an immutable post-departure operational ledger.
* `id`: Autogenerated review record ObjectId.
* `flightNumber`: Key relationship pointer.
* `totalDelayMinutes`: Evaluated duration index.
* `loggedDelays`: Snapshot collection (`List<String>`) capturing turnaround anomalies.
* `timestamp`: Record finalization clock.
